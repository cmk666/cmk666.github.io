发现操作 333，实质上只要找一最大的区间 L≤x≤RL\le x\le RL≤x≤R 使得 ∀i∈[L,R],ci∈A\forall i\in[L,R],c_i\in A∀i∈[L,R],ci​∈A，然后求出 [L,R][L,R][L,R] 的权值和即可，树状数组维护。

Stage 1, D: Pixel Art

statement

给定 n×mn\times mn×m 的格子，初始全是白色，有 kkk 个操作，每个操作会把一条横线段或竖线段染黑（保证染黑的地方不交）。

全部操作完后，对于每个 i∈[1,n]i\in[1,n]i∈[1,n]，求出 1∼i1\sim i1∼i 行的黑格子数，以及黑格子组成的四连通块数。

多测，1≤n,m,k≤105,∑n,∑k≤5×1051\le n,m,k\le10^5,\sum n,\sum k\le5\times10^51≤n,m,k≤105,∑n,∑k≤5×105。

solution

把操作对于左上的点扫描线。

开若干个 set 维护当前行的连通情况，列的连通情况，合并的时候并查集即可。答案是容易维护的。

Stage 1, F: Chaleur

statement

给定一张无向简单图，保证这个图可以划分为一个团和一个独立集。

分别求出：有多少种方案选出一个最大的团；有多少种方案选出一个最大的独立集。

多测，1≤n≤105,0≤m≤105,∑n,∑m≤2×1061\le n\le10^5,0\le m\le10^5,\sum n,\sum m\le2\times10^61≤n≤105,0≤m≤105,∑n,∑m≤2×106。

solution

先考虑怎么弄出划分方式：把所有点按度降序排序，一次加入某个度的所有点，若不是团则这些点里只能加至多一个，且度更小的不能再加，枚举即可。

容易发现，一个团最多只能包含在原划分方案中的独立集中的一个点，反之亦然，直接枚举更新答案。

精细实现可以做到 O(n+m)O(n+m)O(n+m)。

Stage 3, B: Palindromic Beads

statement

给定一颗树，点上有一个权值 cic_ici​，选一条链，使得链上的最长回文子序列最长，求这个长度。

1≤ci≤n≤2×1051\le c_i\le n\le2\times10^51≤ci​≤n≤2×105，一个权值只会出现至多两次。

solution

把两两配对的点按距离从大到小排序，那么可以从 iii 转移到 jjj，当且仅当 jjj 的路径是 iii 的路径的子路径。

把树拍成 dfn 序，那么分 jjj 两个点是否成祖先关系，可以得出 iii 两个点 dfn 序的范围，于是变成了单点改矩形 max⁡\maxmax，树套树即可。

注意判断最中间两个距离 >1>1>1 时可以加一个点形成奇回文，以及答案至少为 111。

时间复杂度 O(nlog⁡2n)O(n\log^2n)O(nlog2n)。

Stage 3, C: Clique Challenge

statement

给定一张无向图，求其团的数量，对 109+710^9+7109+7 取模。

1≤n,m≤1031\le n,m\le10^31≤n,m≤103。

solution

经典 NPC，先往乱搞去想。

把所有点按照度重编号，枚举团中编号最小的点，可能和它在一个团内的只有 2m\sqrt{2m}2m​ 个点，直接 Meet in the middle 做就好了。

精细实现可以做到 O(m⋅22m2)O(\sqrt m\cdot2^{\frac{\sqrt{2m}}2})O(m​⋅222m​​)，当然多带个 m\sqrt mm​ 也跑的飞快。

Stage 3, F: Flying Ship Story

statement

两种操作：加入一个三元组 (x,y,w)(x,y,w)(x,y,w)；给定 x′,y′x',y'x′,y′，查询 x≠x′∧y≠y′x\ne x'\land y\ne y'x=x′∧y=y′ 的三元组 (x,y,w)(x,y,w)(x,y,w) 的 www 的最大值，若无则为 000。

强制在线，1≤q≤106,1≤x,y,w≤1091\le q\le10^6,1\le x,y,w\le10^91≤q≤106,1≤x,y,w≤109，4MB4MB4MB。

solution

考虑维护一个二维平面，一个点的值即为询问的答案。

按 www 降序加入一个三元组 (x,y,w)(x,y,w)(x,y,w)，相当于把除了直线 x=x,y=yx=x,y=yx=x,y=y 之外所有为 000 的地方全部赋值成 www。

维护为 000 的集合是简单的，只有全集、十字、横线、竖线、两个点、一个点、空集 777 种状态，转移也很显然。

当加入一个三元组时，如果没有赋值，那么这个三元组就是废的，直接删掉就好。容易证明有用的三元组只有 ≤4\le4≤4 个，所以只要维护这些三元组就行。

Stage 3, G: GCD of Pattern Matching

statement

给定进制 mmm 和一个模式串 PPP，把模式串中的字母映射到 0∼m−10\sim m-10∼m−1 的数（不能有前导 000），按 mmm 进制转成 101010 进制可以得到若干个数，求它们的 gcd⁡\gcdgcd。

多测，1≤t≤5×105,2≤m≤16,1≤∣P∣≤161\le t\le5\times10^5,2\le m\le16,1\le|P|\le161≤t≤5×105,2≤m≤16,1≤∣P∣≤16。

solution

特判 m=2m=2m=2，此时 P1P_1P1​ 只能映射到 111，另一个只能映射到 000。

否则，设 fcf_cfc​ 表示字符 ccc 的位置的位权之和，那么交换两个字符 x,yx,yx,y 表示的数会产生 fx−fyf_x-f_yfx​−fy​ 的整数倍的贡献。

因此只要随便映射，然后给所有的 ∣fx−fy∣|f_x-f_y|∣fx​−fy​∣ 取 gcd⁡\gcdgcd 即可。

令 ∣P∣|P|∣P∣ 中不同字符有 ccc 个，时间复杂度 O(m+∣P∣+clog⁡c)O(m+|P|+c\log c)O(m+∣P∣+clogc)。

给定序列 a,b,ca,b,ca,b,c，定义函数 fi(x)=ix+max⁡o+p+q=i{ao+bp+cq}f_i(x)=ix+\max_{o+p+q=i}\{a_o+b_p+c_q\}fi​(x)=ix+maxo+p+q=i​{ao​+bp​+cq​}，问哪些 iii 满足 ∀j≠i\forall j\ne i∀j=i，不存在 xxx 使得 fi(x)>fj(x)f_i(x)>f_j(x)fi​(x)>fj​(x)。

Stage 7, J: Range Sets

statement

有 nnn 个集合，qqq 次操作，每次操作形如 [l,r][l,r][l,r] 内的集合加/删一个数，或者询问一个集合的大小。

强制在线，1≤n≤109,0≤q≤1051\le n\le10^9,0\le q\le10^51≤n≤109,0≤q≤105，加入的数为不超过 qqq 的正整数。

solution

对于每个数开一颗 ODT，000 表示不在集合里，111 表示在。再用动态开点线段树维护集合大小。

那么加入相当于把区间 000 变成 111，然后把这些位置在线段树上 +1+1+1 即可，删除同理。查询是简单的。

Stage 7, L: esreveR Order

statement

通信题，第一个人要把 n≤1000n\le1000n≤1000 个 646464 位整数传输给第二个人，他最多可以输出 102410241024 个 646464 位整数；第二个人要通过这些来还原原来的 646464 位整数。

然而，在第一个人传输给第二个人时，每个数都有可能大小端反转，即，设发出的二进制位为 b0∼63b_{0\sim63}b0∼63​，则可能收到 b0∼63b_{0\sim63}b0∼63​，也可能收到 b56∼63b48∼55b40∼47b32∼39b24∼31b16∼23b8∼15b0∼7b_{56\sim63}b_{48\sim55}b_{40\sim47}b_{32\sim39}b_{24\sim31}b_{16\sim23}b_{8\sim15}b_{0\sim7}b56∼63​b48∼55​b40∼47​b32∼39​b24∼31​b16∼23​b8∼15​b0∼7​。你要保证无论如何第二个人能成功还原。

solution

称大小端反转后的 xxx 为 x′x'x′，对于每个原来的数，我们额外维护一个 bit 表示 xxx 和 x′x'x′ 第一个不同位置上 xxx 的那一位的值。这样容易还原 xxx。

然而维护这些 bit 需要约 161616 个 646464 位数，它们也会反转，如法炮制，再搞一个数维护就行，在这个数前面加 888 个 000 后面加 888 个 111 区分前后。

Stage 8, J: The Phantom Menace

statement

给你两组字符串，每组 nnn 个，长度均为 mmm。

你要把每组的字符串按任意顺序拼起来（字符串内部不能重排），使得得到的两个字符串循环重构，构造方案或报告无解。

多测，1≤∑nm≤1061\le\sum nm\le10^61≤∑nm≤106，字符串只含有小写字母。

solution

枚举一组的两个字符串间的拼接点对应另一边的哪个空隙。考虑如何 check。

把每个字符串分成两半，把每个小字符串看作点，两半之间连边，则相当于求一条欧拉回路，满足相邻两条边类型不同，直接做就好了。
